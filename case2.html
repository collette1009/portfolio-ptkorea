생성형 AI 활용 CS 운영 자동화 및 AI 도입 전략 수립

AI 도입 전략은 기획서에 그치지 않아야 한다고 생각합니다. CS 반복 업무의 구조를 분석하고, Claude로 Chrome Extension을 직접 구현해 8단계를 2단계로 축소했습니다. VOC 분류·계약서 분석·보고 자동화에 LLM을 실무 적용하며, AI 도입 필요성을 직접 제안해 비개발 직군 유일 공식 승인을 받은 사례입니다. 기획과 실행을 모두 해본 관점에서의 AI 도입 전략입니다.

반복 CS 업무가 속도만이 아닌 데이터 품질 문제였습니다

문제 1 — CS 반복 입력 피로도와 오입력
대량 취소·반품·재배송 처리 과정에서 동일한 항목을 반복 입력해야 하는 구간이 많았습니다. 처리 피로도와 귀책 분류 오입력이 반복됐고, 잘못된 귀책 입력은 비용 정산·운영 집계까지 꼬이게 하는 구조적 리스크였습니다.
문제 2 — AI 도입 논의 자체가 없었음
Gemini Pro 기본 제공 환경에서 추가 AI 도구 도입 논의가 전혀 없었습니다. CS 운영에 AI를 적용하는 것의 필요성을 인식하는 사람이 없었고, 도입 근거를 만들어 제안하는 구조도 없었습니다.
문제 3 — 반복 판단 업무에 사람 시간이 소비됨
VOC 분류, 계약서 분석, 경영진 보고 데이터 집계 등 AI가 할 수 있는 반복 판단 업무에 많은 시간이 사용되고 있었습니다. 구조적 개선이 필요한 업무에 집중할 여력이 부족한 상태였습니다.

기준을 먼저 만들고, AI로 구조를 바꿨습니다

1. CS 자동화 대상 구조 분석
전체가 아닌 반복 가능한 영역만 자동화
자동화 전에, 어떤 항목이 반복 입력되는지 유형별로 먼저 정의했습니다. 재배송·환불·반품별 입력 항목, 조건별 입력값 기준, 처리 유형에 따른 귀책 분류를 구분했습니다. 최종 확인은 담당자가 하도록 남겨 데이터 예외 상황과 오입력 리스크를 통제하는 구조로 설계했습니다.
2. Claude 활용 Chrome Extension 직접 구현
CS 8단계 → 2단계, 1,151건 단독 처리
Claude를 로직 작성·수정·디버깅 보조로 활용해 비개발자 환경에서 Chrome Extension을 직접 구현했습니다. 재배송 등록 시 반복 입력하던 주요 항목 6개를 단축키 한 번으로 자동 입력, 귀책 기준값을 케이스 유형에 따라 자동 반영해 분류 오류를 구조적으로 차단했습니다. 결과적으로 재배송 6단계→2단계, 반품 8단계→2단계로 축소하며 1,151건을 단독 처리했습니다.
3. LLM 활용 범위 확장 및 AI 도입 전략 수립
VOC 분류·계약 분석·보고 자동화까지 적용
Chrome Extension 외에도 Claude·GPT·Gemini를 VOC 분류, 택배 표준약관 분석, 물류 계약서 교차 검증, 일일 자금집행 리포트 자동 생성에 실무 적용했습니다. AI 도입 필요성과 구체적 활용 방안을 정리해 경영진에게 직접 제안, 비개발 직군 유일 AI 도구 업무비 공식 승인을 받았습니다.
4. AI 기반 경영진 보고 자동화
7억+ 일일 자금집행 자동 분류·발송
경영진이 매일 직접 수백 건 시트에서 당일 집행 건을 찾아 판단하던 구조를, Claude가 자동으로 필수·일반을 분류해 매일 아침 발송하는 시스템으로 전환했습니다. 먼저 발견해 구축하고 경영진에게 제안, 승인 후 정식 운영했습니다. AI를 효율화 도구가 아닌 경영진 의사결정 구조의 일부로 적용한 사례입니다.

근거

1. 재배송 등록 자동입력 설계 (Chrome Extension)
재배송 등록 시 반복 입력하던 주요 항목 6개를 Alt+Shift+D 단축키 한 번으로 자동 입력되도록 설계했습니다. 귀책대상 등 케이스별 기준값을 함께 반영해 분류 오류를 줄였고, 마지막 등록은 직접 확인 후 진행하도록 남겼습니다.
2. 반품 등록 자동입력 설계
반품 등록 시 반복 입력하던 주요 항목 6개를 Alt+Shift+S 단축키로 자동 입력되도록 구성했습니다. 반복 입력 시간을 줄이는 동시에 기준값 오입력을 줄여 데이터 정합성을 높였습니다.
3. 생성형 AI 실무 도구화 및 조직 공식 승인
비개발자 환경에서도 생성형 AI를 로직 작성·수정·디버깅 보조에 실무적으로 활용했고, AI 도입의 필요성과 구체적 활용 방안을 경영진에게 제안해 비개발 직군 중 유일하게 관련 도구를 업무용 비용으로 공식 승인받았습니다.

배운 점

1
AI 도입 전략은 기획서가 아닌 현장 검증으로 설득력을 갖습니다. Chrome Extension을 직접 구현해 8단계→2단계를 보여줬을 때, 조직이 AI 도구 비용을 공식 승인했습니다.
2
자동화의 핵심은 속도가 아닌 판단 구조의 설계입니다. 반복 입력은 자동화하되, 최종 확인은 사람에게 남겨 리스크를 통제하는 구조가 운영 안정성과 데이터 정합성을 동시에 높였습니다.
3
비개발자도 LLM을 실무 파트너로 쓰면 로직 설계와 구현이 가능합니다. 도구의 한계가 아니라 어떤 업무를 어떤 방식으로 위임할지를 정의하는 것이 AI 활용의 핵심이었습니다.

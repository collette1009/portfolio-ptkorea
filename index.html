About

VOC 데이터를 CX 구조 문제로 바꾸는 사람,
AI를 판단 파트너로 쓰는 사람

이커머스 CS 현장에서 같은 문의가 반복될 때, 저는 처리 속도보다 반복의 원인을 먼저 찾았습니다. 프로모션 오인 VOC 173건을 전수 분류해 82%가 동일한 구조적 원인에서 발생한다는 것을 데이터로 증명하고, 개발팀과 협의해 CX 개선으로 실행했습니다.

AI 도구를 실무에 도입한 것도 같은 이유입니다. 더 많이 처리하기 위해서가 아니라, 어떤 업무를 어떤 도구에 위임할지를 구조로 설계하는 것 — 저에게 AI는 효율화 도구가 아니라, 사람이 해야 할 판단을 더 선명하게 만드는 구조의 일부입니다. 비개발 직군 중 유일하게 AI 도구 비용을 업무비로 공식 승인받아 생성형 AI/LLM을 실무 파트너로 직접 운용해왔습니다.

카카오VX에서 400여 개 파트너사 운영 데이터를 기준화했고, 초록마을에서 이커머스 CS 전반을 혼자 설계·운영·개선했습니다. 글로벌 이커머스 플랫폼에서도 데이터를 읽고, 구조를 바꾸고, 실행까지 이어가는 방식으로 기여할 수 있습니다.

CS 문의를 건별로 처리하지 않습니다. 반복되는 패턴을 전수 분류해 구조적 원인을 규명하고, CX 개선 과제로 전환합니다.

AI 도입 전략 실무 검증

기획서에 그치지 않습니다. Claude로 Chrome Extension을 직접 구현해 CS 8단계를 2단계로, AI 전략을 현장에서 바로 검증합니다.

기준이 없으면 직접 만들기

CS 판단 가이드, 운영 기준, SLA 처리 기준 — 없는 것을 발견하면 기다리지 않습니다. 데이터로 설계하고, 경영진 보고로 확정합니다.

실제로 해결한 문제들

VOC 데이터 분석 CX 개선 실행 개발팀 협업

VOC 전수 분류를 통한 CX 구조 문제 발굴 및 개선 실행

프로모션 오인 VOC 173건을 전수 분류해 82%의 동일 구조적 원인을 데이터로 규명하고, 배송 제약 상품 오표시 문제를 VOC 패턴에서 발견해 CTO 재제안·당일 구현까지 연결했습니다. 반복 CS 문의를 건별 처리가 아닌 구조 개선으로 차단한 사례입니다.

일 6~10건→0건 CS 문의 구조적 차단
생성형 AI/LLM 활용 CS 운영 자동화 AI 도입 전략

생성형 AI 활용 CS 운영 자동화 및 AI 도입 전략 수립

Claude로 Chrome Extension을 직접 구현해 CS 반복 프로세스를 8단계에서 2단계로 축소했습니다. 생성형 AI/LLM을 VOC 분류·계약 분석·보고 자동화에 실무 적용하고, AI 도입 비용을 조직에서 공식 승인받은 전략 수립 사례입니다.

8→2단계 CS 자동화 · 비개발 유일 AI 승인
VOC 구조화 CX 입력 설계 데이터 정합성 백오피스 CS 운영

고객 의도 기반 환불·교환 입력 구조 재설계 — AS 재확인 왕복 95%+ 제거

자유 텍스트에 묻혀 있던 고객 의도를 필수 선택값으로 구조화해, CS 재확인 왕복을 95% 이상 제거했습니다. 데이터 수집 단계부터 품질을 설계하고, 중복 필드를 활용하는 스펙으로 개발 범위를 최소화해 빠르게 구현한 사례입니다.

95%+ AS 재확인 왕복 제거 · CX 리드타임 단축
Fail-soft UX 고객여정 분석 MVP 기획 개발팀 협업

장바구니 주문 오류 Fail-soft UX 설계 — 이유를 모른 채 이탈하는 고객을 흐름으로 이어가기

고객이 이유를 모른 채 이탈하는 주문 오류 CX 문제를 발견하고, 기술 제약(API 응답 한계)을 전제로 구현 가능한 MVP로 방향을 전환해 Fail-soft UX를 직접 기획·개발팀 합의한 사례입니다. CS 문의 일 2~3건 → 0건.
